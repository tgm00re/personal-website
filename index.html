ChatRoom - Github

What is it?

ChatRoom is a web application that allows users to create and join "rooms," have live chats with others, and modify their profiles. ChatRoom takes inspiration from applications such as Discord and Slack

Technologies Used

MongoDB, React, Express, Node.JS, Socket.IO, AWS EC2, Nginx, JavaScript, Axios, Bootstrap, HTML/CSS

A problem that arose during creation

During the creation of the application, I would frequently come across the issue of having significant lag if multiple messages were sent in a short period of time. In some cases, my web browser would even shutdown due to running out of memory.

How I fixed that problem and what I learned from it

After utilizing JavaScript's console.log() function to see what functions were called as a user uses the application, I quickly realized that I was making far too many calls to the REST API. To fix this, I significantly reduced the number of API calls while also increasing the amount of data that sockets that were receiving and sending. This change optimized the chatrooms, so users could send and receive any number of messages in short periods without issue.

Otter (Ongoing) - Github

What is it?

Otter is a social media application that allows users to share, manage, and interact with posts and connect with other users. This project takes heavy inspiration from Twitter and is still in development.

Technologies Used

Spring Boot, Spring MVC, Spring Data (JPA), React, Apple JS Framework, Google Identity Services Library , HTML, CSS, JavaScript, Bootstrap, Axios, SQL, Socket.IO

A problem that arose during creation

During the creation of Otter, I found myself repeating my code when I had to add the same features to different pages of the site. This led to a lot of unnecessarily cluttered code and made editing the repeated features extremely time consuming

How I fixed that problem and what I learned from it

To fix this problem, I significantly increased the number of React components being used and increased the modularization of my code. This didn't just decrease the amount of code that I had to write, but also reduced the number of bugs in the application when editing different features of the site. This problem helped me understand the true power of React components and improved my knowledge regarding how React would be used in a large-scale project.

MoonBow Pets - Github

What is it?

MoonBow Pets is a web application created from scratch that includes login and registration, connects to a SQL database, allows users to purchase, view, and manage virtual pets, and allows users to add friends and view their profiles and pets

Technologies Used

SQL, Flask, Python, AWS EC2, HTML, CSS, JavaScript, Bootstrap

A problem that arose during creation

MoonBow Pets was one of the first projects I made that required that I design and develop my own entity relationship diagram (ERD). For this reason, I frequently came across the issue of needing to change my database relationships and refactor my code accordingly.

How I fixed that problem and what I learned from it

While the solution to this problem wasn't solved all at once, it was solved throughout the entirety of the project by creating many different relational diagrams for the database entities that I was working with. This gave me a far better understanding of relationships between entities and the importance of gathering requirements and designing before coding.
